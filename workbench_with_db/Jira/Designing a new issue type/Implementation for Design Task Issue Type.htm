Section C: Creating the Screens

 

Part 1: Create the Create Screen

 

Overview:

 

This screen is used to create a Design Task issue in RIJIRA02.

 

Steps:

 

1)      Click on Administration on RIJIRA02 homepage

2)      Scroll down to Issues and click on Screens under Screens

3)      Click on “+ Add Screen”

4)      Name the screen “Design_Task_create_screen_1_0_0_0”, and click Add

5)      Find the screen you just created in the list, and then configure the screen to include the following fields:

a.       Summary

b.      Due Date

c.       Priority

d.      Description

e.       Component/ Sched. Batch

f.       Resolution Description

g.      ProjectKeywords

h.      Epic Link

6)      Click on Screens in the side nav bar to navigate back to the View Screen page

 

Part 2: Create the Edit Screen

 

Overview:

 

This screen is used to edit a Design Task issue in RIJIRA02.

 

Steps:

 

1)      Click on Administration on RIJIRA02 homepage

2)      Scroll down to Issues and click on Screens under Screens

3)      Find “Design_Task_Create_Screen_1_0_0_0” and click copy.

4)      Change the name of this screen to “Design_Task_Edit_Screen_1_0_0_0”

5)      Click Save

a.       Create Issue

                                                             i.      Choose “Design_Task_Edit_Screen_1_0_0_0” as the screen

 

Section E: Creating the Issue Type Screen Scheme

 

Overview:

 

This scheme is used to choose which screens are shown for each issue operation.

 

Steps:

 

1)      From the Administration homepage, click on Issue Type Screen Schemes under Issues

2)      Click “ Copy” besides software_project_issue_type_screen_scheme_2_0_4_2

3)      Change the name of the scheme to “software_project_issue_type_screen_scheme_2_0_4_3”

4)      Edit description to say “Copy of software_project_issue_type_screen_scheme_2_0_4_2 with Design Task Issue type”

8)      Select “Design Task” and the screen scheme created in Section D for the Screen Scheme, which is “Design_Task_screen_scheme_1_0_0_0” and click Add

9)      Click on Administration to return to the Administration homepage.

2)      Click on Copy besides software_project_workflow_scheme_5_0_0_9.

Section I: Creating the Workflow

 

Overview:

 

These steps will create a custom workflow for the Design Task Issue Type.

 

Steps:

 

Transitions:

 

1)      From the Administration homepage, click on Workflows under Workflows

2)      In the top-right corner, click “+ Add workflow”

3)      Name the workflow “Design_Task_workflow_1_0_0_0” and click Add.

4)      Add the following steps to the workflow: (All linked statuses are the same as the names)

a.       Open

b.      Assigned

c.       In progress

d.      Reopened

e.       Closed

5)      Add the following transitions to the Open Step

a.       In Progress >> In Progress

6)      Add the following transitions to the Assigned Step

a.       Assign to Staff  >> Assigned

                                                              i.      Transition View for transition: assign_to_staff_screen_1_0_0_0

7)      Add the following transitions to the In Progress Step

a.       Resolve and Close >> Closed

                                                              i.      Transition view for transition: Close_resolve_transition_1_1_0_0

8)      Add the following transitions to the Reopened Step

a.       Start Progress >> In Progress

9)      Add the following transitions to the Closed Step

a.       Reopen >> Reopened

                                                              i.      Transition view for transition: Close_resolve_transition_1_1_0_0

 

Conditions: (*MAKE SURE THE CONDITIONS ARE OR, NOT AND)

 

10)  For Assign to Staff, add the following conditions to this transition using Conditions tab:

a.       Click Add, and choose Permission Condition.  Click Add again

b.      Choose Assign Issues, and click Add

 

11)  For Resolve and close, add the following conditions to this transition:

a.       Click Add, and choose Permission Condition.  Click Add again

b.      Choose Assign Issues, and click Add

c.       Click Add to add a new condition to add a subtask condition

d.      Choose Sub-Task Blocking Condition and choose the following conditions:

                                                              i.      Closed, Passed Verification

                                                            ii.      Inspected, Ready for Integration

                                                          iii.      Integrated, Ready for Test

                                                          iv.      In QA Test

                                                            v.      Passed QA, Pre Customer

                                                          vi.      In Customer Test

                                                        vii.      Closed, Duplicate

                                                      viii.      Closed, Not Reproducible

                                                          ix.      Closed, Not a Problem

                                                            x.      Resolved, Ready For Review

                                                          xi.      Closed Rejected

                                                        xii.      Closed.

 

12)  For Start Progress, add the following conditions to this transition:

a.       Click Add to add a new conditions, and choose the Only Assignee Condition

b.      Click Add to add this condition to this transition.

 

Validators:

 

13)  For Assign to Staff

a.       Required Fields -> Choose Assignee

 

14)  For Resolve and Close

a.       Required Fields -> Resolution Description

 

Post Functions:

 

15)  For transition Resolve and Close

a.       Click Add to add a new post function

b.      Choose Update Issue Field and click Add

c.       For Issue Field, choose Resolution

d.      For Field Value, choose Fixed, and click Add

e.       Click Add again to add another post function

f.       Choose Update Issue Custom Field and click Add

g.      For Issue Custom Field, choose Closed_Date

h.      For Custom Field Value, type %%CURRENT_DATETIME%%, and click Add

i.        At the bottom of the Post Function Screen, click edit for the Fire a Generic Event that can be processed by the listeners

j.        Choose Issue Resolved, and click Update

 

16)  For transition Start Progress (For both!)

a.       At the bottom of the Post Function Screen, click edit for the Fire a Generic Event that can be processed by the listeners

b.      Choose Work Started On Issue, and click Update

 

17)  For transition Reopen

a.       Click on Add to add a new post function

b.      Choose Clear Field Value and click add

c.       Choose the field Closed_Date and click Add

d.      Choose Add again to add another post function

e.       Choose Update Issue Field, and click Add

f.       For Issue Field, choose Resolution

g.      For Field Value, choose None

h.      Click Add

 

18)  Return to Section H to finish the remaining steps!

 

Appendices

 

*Once all the above steps have been performed, each project that will use this new issue type must be assigned the new schemes using the below steps.

 

Part 1: Changing the Workflow Scheme

 

Steps:

 

1)      In administration, click on Projects in the top nav bar and find “PROJECT NAME”

2)      In the homepage for “PROJECT NAME”, click on the Workflow Scheme currently associated with the project.

3)      Click on “Switch Scheme”

4)      Choose “software_project_workflow_scheme_5_0_1_0” and click Associate

5)      Go back to the Summary page of the Project.

 

Part 2: Changing the Issue Type Scheme

 

Steps:

 

1)      Under Issue Types, click on the current Issue Type Scheme associated with this project.

2)      Click on Action’s in top-right corner. (You may need to scroll right)

3)      Click on “Use a different scheme”

4)      Choose “software_project_issue_type_scheme_1_1_3_7”

5)      Click Ok

6)      Go back to the Summary page of the project

 

Part 3: Changing the Issue Type Screen Scheme

 

Steps:

 

1)      Under Screens, click on the current Issue Type Screen Scheme associated with this project.

2)      Click on Action’s in the top-right corner

3)      Click on “Use a different scheme”

4)      Choose “software_project_issue_type_screen_scheme_2_0_4_3”

5)      Click Associate

6)      Go back to the Summary page of the project

 

 

 

Part 4: Changing the Field Configuration Scheme

 

Steps:

 

1)      Under Fields, click on the current Field Configuration Scheme associated with this project.

2)      Click on Action’s in the top-right corner.

3)      Choose “Use a different scheme”

4)      Choose “software_project_field_configuration_scheme_2_1_3_6”

5)      Click Associate

6)      Go back to the Summary page of the project.

 

*Repeat the above 4 parts for any projects wishing to use this new issue type.
